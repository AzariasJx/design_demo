# design_demo
设计模式练习demo

策略+简单工厂+门面模式
项目需求：接入多个支付系统进行支付

责任链模式
项目需求：付款完成以后，投放业务，可以在资源位置中展示当前用户的资源
要求 

1. 允许运营人员配置需要展示的资源（配置中心）
2. 资源过滤规则可变
* 过滤规则大部分可重用，但是也会有拓展和变更
* 不同资源位的过滤规则和顺序不一样
* 同一个资源由于业务阶段不同可能规则不一样
3. 允许规则实时增减和顺序调整

过滤规则：
1. 用户个人资质是否满足投放业务
2. 用户所在城市是否在业务投放城市
3. 用户近期所购买的产品是否符合业务投放人群？
4. 新用户首次购买投放指定的业务


单例模式
在我们支付工厂的时候，我们使用了反射进行创建对象，每次调用都会通过反射获取对象会比较影响性能
所以我们使用单例模式，对工厂返回的支付策略进行优化
